# Run PI operations

1. Identify audience, decision, deadline, owner, and source artifacts.
2. Reuse the newest approved template or prior example from Desktop, Downloads, or Documents.
3. For recommendation letters, require the target, candidate CV, relationship, concrete contributions, comparative evidence, and deadline. Never invent achievements.
4. For trainee reviews, separate scientific progress, decisions needed, blockers, commitments, career development, and next checkpoint.
5. For group meetings, build an agenda that surfaces decisions and cross-project dependencies, not only status updates.
6. For teaching, align learning objectives, assessment items, and lecture content; check difficulty and ambiguity.
7. Treat HR, health, performance, immigration, and conflict matters as confidential. Minimize copied personal data.
8. Deliver the artifact plus owners and dated next actions.

Read `references/pi-cadence.md` when designing recurring operating rhythms.
