# Design-make-test-learn checklist

- Design: objective, search space, constraints, candidate rationale, model version
- Make: synthesis route, identity and purity checks, formulation process, batch record
- Test: controls, dose, route, model, randomization, readouts, quality thresholds
- Learn: preprocessing, excluded data, uncertainty, model update, interpretation limits
- Decide: advance, repeat, redesign, stop, or seek orthogonal validation
- Translate: tolerability, biodegradability, manufacturability, stability, scale-up, regulatory relevance

Every round should leave a machine-readable dataset and a short human decision memo. The dataset serves model improvement; the memo preserves scientific judgment and context that the model cannot recover from rows alone.
